# Contributing to Focus

Thank you for your interest in making Focus better! 🎉

Focus is a private, local-first video manager that helps you organize, search, transcribe, and manage your video library securely on your computer. This guide will help you provide feedback that helps us understand and address issues quickly.

## 🐛 Reporting Bugs

Found a bug? We want to know about it! Here's how to write a great bug report:

### Before You Report

1. **Update to the latest version** - Make sure you're using the most recent version of Focus
2. **Search existing issues** - Someone might have already reported it
3. **Try to reproduce** - Can you make it happen again?

### Writing a Bug Report

When creating a bug report, please include:

- **Clear title** - Be specific (e.g., "Transcription fails for MOV files over 1GB" instead of "App doesn't work")
- **Steps to reproduce** - List exactly what you did before the bug occurred
- **Expected vs. actual behavior** - What should have happened? What actually happened?
- **System information**:
  - Operating system and version (e.g., macOS 14.2, Windows 11)
  - Hardware specs (especially for Mac users: M1, M2, M3, M4, or Intel)
  - Focus version number
  - Number of videos in your library (helps us understand performance issues)
- **Video details** (if relevant):
  - Video format (e.g., MP4, MOV, AVI)
  - Video file size and duration
  - Where the video was imported from
- **Logs and errors** - Include any error messages or crash logs if available
- **Screenshots or videos** - Visual aids can be incredibly helpful!

### Where to Find Logs

Logs can help us diagnose issues faster. Here's where to find them:

**macOS:**
- Open Console.app and search for "Focus"
- Or check: `~/Library/Logs/Focus/`

**Windows:**
- Check Event Viewer under Application logs
- Or look in: `%APPDATA%\Focus\logs\`

**Linux:**
- Check: `~/.config/Focus/logs/` or `~/.local/share/Focus/logs/`

*(Note: Adjust paths based on your actual app structure)*

## ✨ Suggesting Features

Have an idea to make Focus even better? We'd love to hear it!

### Before You Suggest

1. **Check existing requests** - Your idea might already be there (you can upvote it!)
2. **Think it through** - Consider how it fits with Focus's purpose and existing features

### Writing a Feature Request

A great feature request includes:

- **The problem** - What are you trying to accomplish? What's frustrating right now?
- **Your proposed solution** - How do you envision this working?
- **Use cases** - Real-world examples of when you'd use this feature
- **Alternatives** - Have you tried other ways to accomplish this?
- **Screenshots or mockups** - If you can visualize it, share it!

### What Makes a Feature Likely to Be Implemented?

Features that are more likely to be considered:

- ✅ Align with Focus's core purpose
- ✅ Benefit many users (not just edge cases)
- ✅ Are clearly described with specific use cases
- ✅ Have community support (upvotes and discussion)

## 💬 General Guidelines

### Be Respectful and Constructive

- Remember there are humans behind this app who care about making it great
- Be patient - we read every issue but may not respond immediately
- Stay on topic and keep discussions focused
- Be open to feedback and discussion

### Quality Over Quantity

- One well-written issue is better than multiple rushed ones
- If you have multiple unrelated issues, create separate reports
- Update existing issues rather than creating duplicates

### After You Submit

- **Watch your issue** - We might ask follow-up questions
- **Test fixes** - If we release a fix, please verify it works for you
- **Close resolved issues** - If your issue is fixed or no longer relevant, let us know!

## 🚀 Thank You!

Your feedback is invaluable in making Focus better for everyone. Every bug report and feature request helps us understand how people use Focus and what they need from it.

We're grateful for your time and effort in helping improve Focus! 🙏
